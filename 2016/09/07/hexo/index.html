HEXO博客的搭建

HEXO + github 搭建自己的博客平台

HEXO是一个简单地、轻量地、基于Node的一个静态博客框架，可以方便的生成静态网页托管在github。
github是一个代码托管平台。由于它的空间免费稳定，可以用于介绍托管在github上的Project或者搭建网站。

第一步 下载 git+nodejs

在使用Hexo之前，有两样东西是必不可少的 ,别问为什么，先去下载，下载完了之后才能进行下面的操作。

  • 下载 git
  • 下载 nodejs

第二步 安装hexo

好了，现在默认你已经下载并成功安装了上边的两个东西。
接下来，随便在电脑什么地方创建一个文件夹 HEXO。

然后双击进去。进去之后，鼠标点击右键，会出现 Git Bash Here ，别怕，点一下，这个时候 「黑窗口」 就会出现。此时，需要你先给电脑上安装Hexo，很简单，在黑窗口下输入如下命令 。

$ npm install -g hexo

注意：-g是指全局安装hexo。
安装完成之后，继续依次输入如下命令，安装依赖包，注意顺序

$ hexo init
$ npm install

回到「HEXO」文件夹下，你会看到，多了好多东西，不要慌，继续依次输入如下命令

$ hexo g（hexo generate）
$ hexo s（hexo server）

不出什么意外的话，在浏览器里输入 http://localhost:4000/
如果能打开，那么恭喜你！你的博客搭建完成了！

###第三步 在github创建Repositor(仓库)###

创建的时候注意Repository的名字。比如我的Github账号是kaHBT，那么我应该创建的Repository的名字是：kaHBT.github.io。

第四步 修改配置文件

到你刚刚创建的Repository下，找到SSH： 如： git@github.com:kaHBT/kaHBT.github.io.git

先点击SSH，然后复制里面的地址。然后编辑我们本地HXMO文件夹的_config.yml文件
deploy:
type: git
res:git@github.com:kaHBT/kaHBT.github.io.git
branch: master

第五步 完成部署

最后一步，快要成功了，键入指令：
$ hexo g
$ hexo d (hexo deploy)

OK，我们的博客就已经完全搭建起来了，在浏览器输入（当然，是你的用户名）：

http://kaHBT.github.io/

###一些常用命令：

hexo new”postName” #新建文章

hexo new page”pageName” #新建页面

hexo generate #生成静态页面至public目录

hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）

hexo deploy #将.deploy目录部署到GitHub

hexo help # 查看帮助

hexo version #查看Hexo的版本

###报错总结

ERROR Deployer not found: git 或者 ERROR Deployer not found: github

解决方法： npm install hexo-deployer-git –save

ERROR Process failed: layout/.DS_Store , 那么进入主题里面layout和_partial目录下，使用删除命令：

rm-rf.DS_Store

ERROR Plugin load failed: hexo-server

解决方法，执行命令：

sudo npm install hexo-server

执行命令hexo server，提示：Usage: hexo ….

解决方法，执行命令：

npm install hexo-server –save

再输入网址http://localhost:4000O/终于可以看到属于你自己的blog啦，😄，虽然很简陋，但好歹有了一个属于自己的小窝了。
